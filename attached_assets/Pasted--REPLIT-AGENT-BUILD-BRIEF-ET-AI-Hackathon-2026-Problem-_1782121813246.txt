## 1. APP OVERVIEW

**App Name:** VayuSense — Urban Air Intelligence Command Centre

**One-line description:** A real-time AI-powered air quality intelligence dashboard that fuses CAAQMS sensor data, weather forecasts, and geospatial source mapping to give city administrators and citizens predictive pollution alerts and evidence-backed enforcement recommendations.

**Core problem it solves:** India's 900+ air quality monitoring stations produce data that sits in dashboards nobody acts on. City administrators lack the intelligence layer to connect sensor readings to specific pollution sources, predict tomorrow's AQI before it becomes dangerous, and deploy inspectors to the right locations. VayuSense closes that gap.

**Target users:**
- Primary: City pollution control officers / municipal administrators
- Secondary: Citizens checking ward-level health risk

## 2. FULL FEATURE LIST

### CORE (must have — judges will look for these)

1. **Live AQI Dashboard** — Real-time AQI cards for 6 Indian cities (Delhi, Mumbai, Kolkata, Bengaluru, Chennai, Pune) with colour-coded severity bands (Good / Satisfactory / Moderate / Poor / Very Poor / Severe)
2. **Interactive Geospatial Map** — Leaflet.js map showing monitoring stations, pollution hotspot heatmap overlay, and clickable markers with station details
3. **AI Predictive Forecasting Panel** — 72-hour ward-level AQI forecast using a simple ML model (polynomial regression or LSTM simulation on synthetic data); displays as a chart with confidence band
4. **Pollution Source Attribution Engine** — AI agent that analyses current AQI + wind direction + land-use data and attributes pollution % by source category (vehicles, construction, industries, crop burning, dust) — displayed as an animated donut chart
5. **Enforcement Intelligence Module** — Ranked list of enforcement action recommendations with source type, GPS coordinates, confidence score, and a "Generate Notice" button that produces a downloadable PDF notice
6. **Citizen Health Advisory Panel** — Ward-level health risk score, vulnerable population mapping (hospitals/schools overlay on map), and advisory text generated via Claude API in English + one regional language toggle (Hindi)
7. **Multi-City Comparison View** — Side-by-side AQI trend line charts across all 6 cities for the past 30 days

### NICE-TO-HAVE (bonus points with judges)

8. **AI Chat Copilot** — Sidebar chat powered by Claude API where administrators can ask questions like "Which zone will breach AQI 300 tomorrow?" and get cited, data-backed answers
9. **Intervention Effectiveness Tracker** — Simulated before/after AQI improvement chart after a mock intervention is "applied"
10. **Alert Subscription Panel** — Form to subscribe to ward-level alerts (stored in local state; no real email sending needed for demo)
11. **Animated Data Ingestion Feed** — Live scrolling ticker showing simulated sensor readings arriving in real time

## 3. TECH STACK

**Frontend:**
- React (Vite scaffold)
- Tailwind CSS for styling
- Recharts for line/bar/donut charts
- Leaflet.js + React-Leaflet for the map
- Framer Motion for animations

**Backend:**
- Node.js + Express (single lightweight server)
- In-memory data store (no database needed; all data is mock/seeded)

**AI Integration:**
- Anthropic Claude API (`claude-sonnet-4-6`) via the `/v1/messages` endpoint for:
  - Generating health advisories in regional languages
  - Powering the AI Chat Copilot
  - Generating enforcement notice text

**APIs (free, no auth or mock-friendly):**
- **OpenWeatherMap Free Tier** — `https://api.openweathermap.org/data/2.5/weather?q={city}&appid={key}` — for real wind speed/direction to feed source attribution logic. Use API key from env var `OPENWEATHER_KEY`. If unavailable, fall back to hardcoded mock weather objects.
- **WAQI (World Air Quality Index) API** — `https://api.waqi.info/feed/{city}/?token={token}` — for real AQI readings. Free token at waqi.info. Use env var `WAQI_TOKEN`. Fall back to mock data if token absent.

## 5. UI & DESIGN INSTRUCTIONS

### Design Philosophy
Industrial command-centre aesthetic — this is a tool for administrators making enforcement decisions, not a consumer wellness app. Dark, data-dense, authoritative. The signature element: a **full-width animated AQI gradient bar** at the very top of every city page that pulses from the current AQI colour (green → hazardous maroon) and serves as an ambient environmental indicator.

### Colour Palette
```
--bg-primary:     #0A0E1A   (deep navy-black, main background)
--bg-surface:     #111827   (card/panel background)
--bg-elevated:    #1C2333   (modals, sidebars)
--border:         #2D3748   (subtle dividers)
--accent-cyan:    #00D4FF   (primary interactive, chart lines, links)
--accent-amber:   #F59E0B   (warnings, moderate AQI)
--accent-red:     #EF4444   (severe AQI, urgent alerts)
--accent-green:   #10B981   (good AQI, positive trends)
--text-primary:   #F1F5F9
--text-secondary: #94A3B8
--text-muted:     #4B5563
```

### AQI Severity Colours (use consistently everywhere)
```
Good (0-50):         #10B981 (green)
Satisfactory (51-100): #84CC16 (lime)
Moderate (101-200):  #F59E0B (amber)
Poor (201-300):      #F97316 (orange)
Very Poor (301-400): #EF4444 (red)
Severe (401+):       #7C3AED (purple)
```

### Typography
- **Display / headings:** `Space Grotesk` (Google Fonts) — technical, modern, slightly geometric
- **Body / data:** `Inter` — neutral, high readability at small sizes
- **Monospace / sensor data values:** `JetBrains Mono` — for AQI numbers, coordinates, timestamps

### Layout
- Max content width: 1440px, centered
- Sidebar: 240px fixed, collapsible on mobile
- Cards: 12px border-radius, 1px border in `--border`, subtle `box-shadow: 0 0 0 1px rgba(0,212,255,0.05)`
- No gradients except the signature top bar and chart fills
- All charts: dark background, no gridlines except faint horizontal lines in `--border` colour

### Animations
- **Signature top bar:** CSS `@keyframes pulse-aqi` — slowly oscillates opacity between 0.7 and 1.0
- **AQI gauge:** SVG arc that animates from 0 to current value on page load (500ms ease-out)
- **Map heatmap:** Opacity fade-in on load (300ms)
- **Donut chart:** Framer Motion `animate` on each slice, staggered 100ms apart
- **Chat messages:** Slide up from bottom with opacity fade (150ms)
- No parallax or scroll-triggered animations (keeps performance clean)

### UI Components
- Custom `<AQIBadge>` component: pill shape, AQI value + label, coloured by severity
- Custom `<StatCard>` component: icon + metric + delta indicator
- `<HotspotTable>` with alternating row shading and a pulsing red dot for critical rows
- Modal for enforcement notice generation with a "Copy to Clipboard" and "Print" button
- Toast notifications (bottom-right) for simulated real-time alerts

function generateForecast(baseAQI) {
  return Array.from({ length: 72 }, (_, i) => ({
    hour: i,
    aqi: Math.round(baseAQI + Math.sin(i / 8) * 30 + (Math.random() - 0.5) * 20),
    lower: Math.round(baseAQI + Math.sin(i / 8) * 30 - 15),
    upper: Math.round(baseAQI + Math.sin(i / 8) * 30 + 15)
  }));

### API Integration Details

**Claude API (Anthropic):**
- Endpoint: `POST https://api.anthropic.com/v1/messages`
- Model: `claude-sonnet-4-6`
- Max tokens: `1000`
- No API key needed in client code (handled by Replit secrets — store as `ANTHROPIC_API_KEY`)
- Use for:
  1. Health advisory generation: Prompt = `"Generate a 3-sentence public health advisory for {cityName} with current AQI {aqi}. Include specific precautions for vulnerable groups. Then translate to Hindi. Format as JSON: {english: '...', hindi: '...'}"`
  2. Chat copilot: Full conversation history in messages array; system prompt = `"You are VayuSense, an AI air quality analyst for Indian cities. You have access to current AQI, pollutant levels, and 72-hour forecasts. Answer administrator questions concisely with data references. Current city data: {JSON.stringify(cityData)}"`
  3. Enforcement notice: Prompt = `"Generate a formal municipal enforcement notice for: Zone: {zone}, Source Type: {sourceType}, Current AQI: {aqi}. Include regulatory references to CPCB norms. Keep under 200 words."`

**OpenWeatherMap (optional enhancement):**
- Endpoint: `GET https://api.openweathermap.org/data/2.5/weather?q={city},IN&appid={OPENWEATHER_KEY}&units=metric`
